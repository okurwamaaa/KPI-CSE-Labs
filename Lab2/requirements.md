# Requirements & Traceability Matrix

## 1. Список вимог

### Функціональні вимоги (FR)

**FR1:** Система повинна надавати повний каталог товарів, що включає назву, ціну, зображення, опис та категорію для кожного артикула.
**FR2:** Система повинна забезпечувати механізм реєстрації та авторизації користувачів (як покупців, так і адміністраторів) з підтримкою відновлення пароля.
**FR3:** Система повинна надавати основний робочий простір у вигляді вітрини товарів, яку можна фільтрувати за категорією, ціною та назвою (UC1).
**FR4:** Система повинна підтримувати повний життєвий цикл замовлення: від додавання в кошик до зміни статусу ("New", "Processing", "Shipped", "Delivered").
**FR5:** Детальна сторінка товару повинна агрегувати та відображати всю пов'язану інформацію: галерею фото, технічні характеристики, ціну та кнопку додавання в кошик.
**FR6:** Система повинна підтримувати застосування промокодів та знижок, автоматично перераховуючи загальну вартість кошика (UC6).
**FR7:** Користувачі повинні мати змогу оформити замовлення (Checkout), вказавши адресу доставки та обравши метод оплати.
**FR8:** Адміністратор повинен мати інструменти управління каталогом, що включають можливість додавання, редагування та видалення товарів (UC7).
**FR9:** Адміністратор повинен мати змогу змінювати статус замовлень та переглядати інформацію про покупця для організації доставки.
**FR10:** Система повинна автоматично надсилати сповіщення на email користувача при зміні статусу замовлення або успішній оплаті (UC8).
**FR11:** Система повинна інтегруватися із зовнішньою платіжною системою для безпечної обробки транзакцій (UC5).
**FR12:** Робочий сценарій Покупця (Guest/User) включає: знайти товар, переглянути деталі, додати в кошик, оформити замовлення та оплатити.
**FR13:** Робочий сценарій Адміністратора (Admin) включає: управління товарами, перегляд нових замовлень, зміну статусів та контроль оплати.
**FR14:** Управління даними користувача має бути захищеним: історія замовлень доступна лише авторизованому користувачеві.

### Нефункціональні вимоги (NFR)

**NFR1 (Завантаження сторінок):** Сторінка каталогу (UC1) та сторінка товару повинні повністю завантажуватись не довше ніж за 2.5 секунди.
**NFR2 (Продуктивність пошуку):** Застосування фільтрів до каталогу товарів повинно повертати результат не довше ніж за 1.5 секунди, навіть при обсязі бази до 100,000 товарів.
**NFR3 (Одночасні користувачі):** Система повинна стабільно підтримувати до 1000 одночасно активних покупців, зберігаючи час відповіді UI у вказаних межах.
**NFR4 (Безпека та доступ):** Кожен запит до API, що стосується персональних даних або оплати, повинен проходити через захищений протокол HTTPS та вимагати валідного токена.
**NFR5 (Доступність системи):** Система повинна мати показник доступності (Uptime) 99.9% протягом періодів розпродажів та активного трафіку.
**NFR6 (Обробка зображень):** Система повинна підтримувати завантаження зображень товарів розміром до 5 МБ та автоматично оптимізувати їх для веб-перегляду.
**NFR7 (Цілісність даних):** Операції списання коштів (UC5) повинні бути транзакційними: гроші списуються тільки при успішному створенні замовлення.
**NFR8 (Сумісність):** Веб-інтерфейс магазину повинен коректно відображатися на мобільних пристроях (Responsive Design) та в останніх версіях Chrome, Safari, Firefox.

## 2. Use Cases

**UC1:** Search Products (Пошук товарів)
**UC2:** Register / Login (Реєстрація та вхід)
**UC3:** Add to Cart (Додавання в кошик)
**UC4:** Place Order (Оформлення замовлення)
**UC5:** Process Payment (Обробка оплати)
**UC6:** Apply Discount (Застосування знижки)
**UC7:** Manage Products (Управління товарами)
**UC8:** Send Notifications (Відправка сповіщень)
